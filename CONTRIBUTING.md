# Contributing Guidelines

Thank you for your interest in contributing to this project!

## How to Contribute

1. **Fork the repository** - Create a personal fork of the project
2. **Create a branch** - Use a descriptive branch name:
   - `feature/add-new-functionality`
   - `fix/resolve-issue`
   - `docs/update-documentation`

3. **Make your changes** - Ensure your changes are well-documented and tested

4. **Commit messages** - Use clear, descriptive commit messages:
   ```
   git commit -m "Add feature: description of change"
   ```

5. **Push to your branch** - Push changes to your fork
6. **Create a Pull Request** - Submit a PR with a clear description of changes

## Guidelines

- Keep commits focused on a single feature or fix
- Document any significant changes
- Test your changes before submitting
- Follow existing code style and conventions
- Add comments for complex logic
- Update README if adding new features

## Code Standards

- Use meaningful variable and function names
- Comment complex algorithms
- Keep functions modular and reusable
- Test edge cases

## Issues

- Check existing issues before creating a new one
- Provide clear descriptions of bugs or feature requests
- Include steps to reproduce for bug reports

## Questions?

If you have questions about contributing, please open an issue or contact the maintainers.
